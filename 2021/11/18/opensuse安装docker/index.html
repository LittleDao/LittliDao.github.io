opensuse安装docker

  • 作者 Little Dao
  • ~2.77K 字
本文最后编辑于 前，其中的内容可能需要更新。

在opensuse下安装docker

0x000 软件版本

  1. opensuse leap 15.3
  2. VMware 16Pro

0x001 配置镜像源

这部分可以配置也可以不配置。如果需要配置的话，选择一个自己喜欢的镜像源并查询取配置吧。
我这里使用的是中国科学技术大学开源软件镜像，在镜像列表中搜索opensuse，后面有个help可以参考。
如果你懒得打开我这里将其中的代码部分展示出来。

除命令配置外，也可以用图形界面来配置，软件源的配置在YaST里，图形界面这块就不在描述，有需要可以去[help](http://mirrors.ustc.edu.cn/help /opensuse.html)里查看。

0x002 安装docker

这里通过命令来安装，参照官网的维基来进行配置。

0x003在docker下安装mariadb

既然docker已经安装好了，那就装个数据库试试吧。

dockers还有些常用的命令，这里记录一下
docker start mariadb 启动mariadb，其中mariadb为别名，也可以是容器ID
docker stop mariadb 停止mariadb，和启动相似
docker rmi -f IMAGE_ID 清理并删除docker镜像，其中IMAGE_ID为docker images查出来的ID

0x004 一些坑

在我装好mariadb后，我准备在宿主机上进行连接
mysql -uroot -proot，随后就报错，说没有sock文件
后来查了下，docker下需要带host，即mysql -h127.0.0.1 -uroot -proot，这样才行
可当我这么执行后，出现了长时间无响应，最后说让我确认ip是否正确(连接超时等，具体的忘记了)。
我首先怀疑的是防火墙，在YaST里看了下，多了一个docker0。然后看了看也没看懂。就退出去了。
后来还是一直都不行，又检查了防火墙。在里面看到docker0后面有个区域，区域选的是docker。而docker区域的允许里面，并没有值。然而我也不知道添加哪些，就全部添加过去了。保存后，就一直弹窗，说xxxx找不到，是因为我把所有的服务都选了。它将没有的服务都提示了一变。
然后我在运行docker的时候，发现报错。提示我服务没启动。然后我使用命令systemctl start docker.service来启动，也报错。百思不得其解。

翌日，我将虚拟机重启了，docker莫名其妙的就能用了!!
这时候我又在想。为啥？
看了看防火墙，没有变化。看了看服务，已经启动了。
emmm.完全不知道为啥。

bing一下.得出的结论是：

我修改了防火墙规则，但并没有重启防火墙，于是docker的服务就出问题了
然后我重启了虚拟机，防火墙相当于重启了，docker服务也就正常了
至于最开始docker服务正常，防火墙没动，依旧连不上docker的数据库，我想是因为防火墙的规则问题。
我将防火墙的docker规则删除了一些，保留了其中的docker的两个服务。以及一下在public里的服务。其他的我都去掉了，依旧可以使用。

结束

至此，opensuse+docker+mariadb的旅途结束。其中docker run mariadb这里还有很多可以探索的地方，如：指定用户和密码，指定映射的目录。如果需要的话，可以自己查查。
